# GIDSAM UI SYSTEM — CHECKLIST (Design System v1.1)

Этот файл — обязательный чеклист приёмки для проекта «ГИДСАМ».
Любой блок, витрина, страница, компонент, модуль или интерфейс считается ГОТОВЫМ только если пройдены ВСЕ пункты.

Если хотя бы один пункт нарушен — код НЕ ПРИНИМАЕТСЯ.

---

## 0) Обязательная привязка к системе

- [ ] Используются токены ТОЛЬКО из `TOKENS.md` (цвета/тени/радиусы/типографика/переходы).
- [ ] Используются структуры ТОЛЬКО из `COMPONENTS.md`.
- [ ] Лэйауты и контейнеры соответствуют `LAYOUTS.md` (если применимо).
- [ ] Никаких “своих” переменных, теней, радиусов, шрифтов, размеров.

---

## 1) Сетка (самое важное)

- [ ] Везде минимум 2 колонки (даже вакансии, новости, табличный контент, вилки).
- [ ] Списки карточек завернуты в `.gidsam-grid`.
- [ ] Сетка строго по брейкпоинтам:
  - Mobile: 2 колонки, gap 10px
  - ≥768px: 3 колонки, gap 12px
  - ≥1024px: 5 колонок, gap 12px
- [ ] Нет локальных “сломов” сетки (1 колонка, другие gap, другие колонки).

---

## 2) Карточки и изображения

- [ ] Карточки собраны через `.gds-card` (структура как в `COMPONENTS.md`).
- [ ] Карточка кликабельна целиком (не только заголовок/картинка).
- [ ] Изображение всегда в контейнере `.gds-img` + один из aspect ratio классов:
  - `.gds-ar-1x1` / `.gds-ar-4x3` / `.gds-ar-16x9` / `.gds-ar-16x6` / `.gds-ar-9x16`
- [ ] `img` всегда `object-fit: cover`, карточки одного типа не “прыгают” по высоте.
- [ ] Переполнение текста контролируется (1 заголовок + 1–2 строки мета, без резины).

---

## 3) Цвета, радиусы, тени

- [ ] Цвета берутся только из переменных `--g-*` (никаких “случайных” HEX).
- [ ] Радиусы только из набора: 12 / 10 / 16 / 8 (или токены `--g-r-*`).
- [ ] Тени только из набора `--g-shadow-*` и `--g-shadow-blue`.
- [ ] Hover-эффекты не ломают компоновку, тени аккуратные iOS-стиля.

---

## 4) Типографика

- [ ] Шрифт: только `--g-font`.
- [ ] Размеры/веса соответствуют токенам (page title / card title / body / meta / badge / button).
- [ ] Нет “микрошрифтов” ради влезания — вместо этого обрезка/2 строки/ellipsis.

---

## 5) Интерактив (кнопки, фильтры, формы)

- [ ] Кнопки только через `.gds-btn` (primary/secondary).
- [ ] Таблетки только через `.gds-pill` (обычная/active/ghost).
- [ ] Инпуты/селекты только `.gds-input`, `.gds-select`.
- [ ] У интерактивных элементов есть понятное hover/focus состояние.
- [ ] На мобиле функционал не зависит от hover (touch-first).

---

## 6) Состояния UI (обязательно)

Каждый список/виджет/лента обязаны иметь:

- [ ] `loading` состояние (видимое + текстом).
- [ ] `empty` состояние (видимое + подсказка).
- [ ] `error` состояние (видимое + текстом).
- [ ] Состояния не “ломают” сетку и не прыгают по высоте.

---

## 7) Доступность (минимум)

- [ ] Если у кнопки/иконки нет текста — есть `aria-label`.
- [ ] Фокус всегда видимый (outline 2px + ring из токенов).
- [ ] Контраст текста читаемый (особенно meta).

---

## 8) Технические запреты (строго)

- [ ] Нет inline-styles.
- [ ] Нет уникальных “левых” классов вне системы без обновления системы.
- [ ] Нет копипасты CSS “как попало” — только через токены/примитивы.
- [ ] Нет зависимостей от конкретных ID блоков (только классы и scope).

---

## 9) Финальная проверка перед коммитом

- [ ] Открылось на мобиле (360–430px) без горизонтального скролла.
- [ ] Открылось на таблетке (768px) и десктопе (≥1024px) — сетка соответствует.
- [ ] Карточки не прыгают по высоте, изображения обрезаются корректно.
- [ ] Все интерактивы кликаются, фокус виден, состояния loading/empty/error есть.
