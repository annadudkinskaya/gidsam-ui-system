# GIDSAM UI SYSTEM — CONVENTIONS (Design System v1.1)

Этот файл фиксирует ИМЕНА, ПРАВИЛА и СОГЛАШЕНИЯ проекта «ГИДСАМ».
Все файлы, классы, компоненты и сущности именуются ТОЛЬКО по этим правилам.
Любые отклонения запрещены.

---

## 1) Имена файлов (в корне репозитория)

Все файлы лежат В ОДНОМ КОРНЕ, без папок.

Обязательный набор:

- PROMPT.md — философия и закон системы
- TOKENS.md — дизайн-токены (цвета, радиусы, тени, типографика)
- COMPONENTS.md — канонические HTML-структуры
- LAYOUTS.md — правила компоновки страниц и секций
- CHECKLIST.md — чеклист приёмки
- CONVENTIONS.md — этот файл
- EXAMPLES.md — примеры корректного использования
- STYLES.md — базовые CSS-примитивы (если нужен отдельный файл)

Других файлов быть не должно без обновления этого списка.

---

## 2) Имена CSS-классов

### 2.1 Префиксы

- `gds-` — базовые UI-примитивы (card, btn, img, input)
- `gidsam-` — системные контейнеры и layout-логика
- `is-` — состояния (is-active, is-loading)
- `has-` — модификаторы наличия (has-image)

Другие префиксы запрещены.

---

### 2.2 Формат имен

Только kebab-case.

ПРАВИЛЬНО:
- `gds-card`
- `gds-card-title`
- `gds-btn--primary`
- `gidsam-grid`

ЗАПРЕЩЕНО:
- camelCase
- snake_case
- сокращения без смысла
- абстрактные названия (`block1`, `item-new`)

---

### 2.3 Модификаторы

Модификаторы ТОЛЬКО через `--`.

Пример:
- `gds-btn--primary`
- `gds-pill--active`
- `gds-card--featured`

Никаких вложенных вариаций классов.

---

## 3) HTML-структура

- Один компонент = одна структура
- Один тип карточек = одна HTML-схема
- Запрещены вариации «почти такие же»

Если структура отличается — это НОВЫЙ компонент и он ОБЯЗАН быть описан в `COMPONENTS.md`.

---

## 4) CSS-правила

- Никаких inline-styles
- Никаких ID-селекторов
- Никаких `!important`
- Все значения — через токены из `TOKENS.md`
- Все hover/focus/active — через стандартные состояния

---

## 5) JS-интеракции (общее правило)

- JS не меняет стили напрямую
- JS работает через классы (`is-active`, `is-loading`)
- JS не ломает сетку и размеры

---

## 6) Коммиты (единый стиль)

Формат commit message:

- `ui: add tokens`
- `ui: add components`
- `ui: add layouts`
- `ui: add checklist`
- `ui: update components`
- `ui: fix grid rules`

Без эмоций. Без описаний «поправила», «попробовала».
Только факт изменения.

---

## 7) Запреты

Запрещено:

- «Чуть подправить, потому что так красивее»
- Добавлять классы без описания в системе
- Делать исключения «только тут»
- Ломать mobile-first
- Делать одну колонку

---

## 8) Главное правило

Если чего-то НЕТ в системе — ЭТОГО НЕ СУЩЕСТВУЕТ.

Сначала обновляется система.
Потом пишется код.
